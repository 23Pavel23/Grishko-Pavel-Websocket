Отчет по результатам лабораторной работы

1.Информация о системе
ОС: Ubuntu 22.04.5 LTS (Jammy Jellyfish). Ядро: Linux 5.15.0-170-generic (x86_64). CPU: 4 виртуальных процессора (хостовые AMD Ryzen 7 3700X). Память: Всего ~3.9GB RAM (использовано 9%), Swap 2.7GB (не используется). Диски: Основной корневой раздел (/dev/mapper/ubuntu--vg-ubuntu--lv) на 14GB (использовано ~46%), отдельный /boot на 2.0GB. Виртуальная машина настроена с 30GB диском.

2.Сеть
IP-адрес: 192.168.66.132/24 (интерфейс ens33, NAT в VMware). Открытые порты: Порт 53 (DNS) - слушает systemd-resolve на локальном адресе 127.0.0.53. Порт 22 (SSH) - слушает sshd на всех адресах (0.0.0.0 и ::).

3.SSH
Состояние: Сервис ssh активен (running). PID: 2014 (главный процесс sshd). Порт: 22 (как для IPv4, так и для IPv6). Автозапуск: Включен (enabled), т.е. SSH стартует автоматически при загрузке системы.

4.Пользователи и группы
Системные пользователи: root (UID 0) - администратор. Другие системные пользователи не перечислены, но подразумеваются (для служб). "Настоящие" пользователи: grishko (UID=1000) - основной пользователь, создан при установке; boardy (UID=1001) - создан позже, добавлен в группу sudo. Группы: Основная группа для пользователя boardy - boardy, дополнительная - sudo (даёт права администратора).

5.Файловая система
Корневая структура (/) стандартна для Linux. Ключевые каталоги: /bin, /sbin, /lib - ссылки на каталоги внутри /usr; /boot - файлы загрузчика и ядро; /etc - конфигурационные файлы системы; /home - домашние каталоги пользователей (grishko, boardy); /root - домашний каталог пользователя root; /tmp - временные файлы (доступны всем); /var - изменяемые данные (логи, кэш, очереди); /usr - пользовательские программы и библиотеки.

6.Права доступа
Права на директории: / (drwxr-xr-x) - root может всё, остальные читать и переходить; /tmp (drwxrwxrwt) - все могут создавать/удалять свои файлы, но не чужие (бит "sticky"). Права на файлы (chmod): Числовой формат (например, 755) - это сумма прав для владельца, группы и остальных. 4 (чтение), 2 (запись), 1 (исполнение). rw-rw-r-- (664) - владелец и группа читают/пишут, остальные только читают. rwxr-xr-x (755) - владелец может всё, группа и остальные читают/исполняют. rw------- (600) - только владелец может читать и писать.

7.Пакеты и сервисы 
Пакеты: Список установленных пакетов не показан, но система содержит типовой набор (snapd, multipath-tools и т.д.), есть обновления (66 пакетов). 
Запущенные сервисы (основные): ssh, systemd-*, cron, dbus, snapd, open-vm-tools (интеграция с VMware), udisks2 (управление дисками).

8.Конвейеры -
ps aux --sort=-%mem | head -n 11: ps aux - выводит список всех процессов; --sort=-%mem - сортирует их по убыванию использования памяти;head -n 11 - показывает первые 11 строк (заголовок + топ-10 процессов), что мы и видим на скриншотах.
ps -ef | grep ssh: ps -ef - выводит список всех процессов в полном формате; grep ssh - фильтрует строки, оставляя только те, что содержат "ssh". Это стандартный способ найти PID интересующего процесса.
